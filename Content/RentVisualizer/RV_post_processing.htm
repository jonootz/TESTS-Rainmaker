Rent Visualizer: Further Post Processing Stage

The final stage results in what is referred to as the recommended rent before any amenities or pricing offsets are applied. Once the amenities and/or any offsets are considered, then this value should match the Base rent on the New Pricing report. This post optimization stage applies any rounding rules that are in effect. It also applies any lease expiration management premiums or Rec Gen premiums that have been configured.

Inputs

Input

Definition

Rec Gen Used

Indicates whether or not Rec Generation has been set up for this community.

Rec Gen Lease Term

The corresponding lease term as set up in Rec Generation under Admin>Rec Generation (if Rec Gen Used = Y).

Rec Gen Base Lease Term

Displays the lease term pricing that the actual lease term is being based off of according to how Rec Generation has been set up (if Rec Gen Used = Y).

Rec Gen Premium

The additional percentage amount that will be added on to the recommended rent for that lease term(if Rec Gen Used = Y).

Recommended Base Rent

The final rent recommendation for the lease term not including any pricing offsets or amenities.

Recommended Concession

The amount of any concessions being applied. This is configured as a total dollar amount or number of months free under Configuration>Recommendation Strategy.

Recommended Effective Base Rent

The value of the base rent after including any concessions.

Rounded Recommended Rent

The amount of the final rent recommendation after applying the rounding rules in place. The default value is to the nearest $5.

Rounded Concession

The amount of any concessions in place after the rounding rule has been applied.

Base Rent Rounding Parameter

The value of the base rent rounding rule that is currently in place. It can only be changed by LRO Support.

Concession Rounding Parameter

The value of the concession rounding rule that is currently in place. It can only be changed by LRO Support.

Ceiling Type

In this stage, recommended rents are checked to ensure they are within a certain range of the current rents. Ceiling Type defines the range type which can be either dollars or a percentage. This is configured under Configuration>Recommendation Strategy.

Ceiling Value

In this stage, recommended rents are checked to ensure they are within a certain range of the current rents. Ceiling Value is the upper bound for the range check. This is configured under Configuration>Recommendation Strategy.

Minimum Threshold

The Min Change In Rent Week 0 and Week 12, when non-zero, defines a dollar or percentage threshold below which rents for the UT are not written to the rent recommendations table. In this case, LRO will use the rents from the prior post date (Week 0 refers to the current pricing week). This is configured under Configuration>Recommendation Strategy.

Maximum Threshold

The Maximum Threshold is the Min Change In Rent Week 12 configured under Configuration>Recommendation Strategy. When non-zero, this defines a dollar or percentage threshold below which rents for the UT are not written to the rent recommendations table. In this case, LRO will use the rents from the prior post date. Week 12 here actually means the last week of the optimization horizon, as configured in the Optimization Horizon parameter (parameter 1). This Maximum Threshold is used along with the Minimum Threshold defined above to derive the threshold used to determine if recommendations are written to the rent recommendation table for each pricing week from the current week to the end of the pricing horizon.

Current Base Rent

Current Base Rent is populated when the PMS in place is MRI, or specific configurations are in place to populate the LRO current rents table. These current rents can be used as a form of price smoothing called smoothing over post dates. This is different from smoothing over pricing weeks, but has the same goal of reducing price volatility. It uses percentages of pricing across post dates as opposed to pricing weeks.

Current Rent

These current rents can be used as a form of price smoothing called smoothing over post dates. This is different from smoothing over pricing weeks, but has the same goal of reducing price volatility. It uses percentages of pricing across post dates as opposed to pricing weeks.

Current Concession

Current Concession is populated when the PMS in place is MRI, or specific configurations are in place to populate the LRO current rents table.
