Rent Visualizer: Reference Rent Stage

This is the starting point in the pricing process. This stage looks at the competitor rent information that is in place for each competitor, along with the weight/position assigned to that competitor and the lease term category premium in place, to calculate the market composite when the DRRC task runs for its scheduled run date. During this same process, LRO looks at five conditions- exposure level, leasing velocity level, comps increasing, premium current, and comps above/below- to determine a business formula to use when generating a new reference rent value.

Inputs

Input

Definition

Premium LTC

The Lease Term Category Premium which is used in building the market composite for each lease term category. It is configured under Configuration> LTC.

Market Composite

Calculated based on the competitor rents, weights, and positions, which are entered under the Competitors tab (Comp Rents, Comp Weights, Comp Positions).

Last Market Composite

The market composite value that was in place as of the DRRC run before the most recent one.

Current Reference Rent

The value that the reference rent will be the next time DRRC runs.

Override Reference Rent

The reference rent value being used if a dollar amount was manually entered in the green Ref Rent Override box on the Reference Rent screen and the DRRC was run.

Last Reference Rent

The value of the reference rent that is currently being used as of the last DRRC run. New pricing is based off of this amount.

Current Market Composite User

The market composite amount that was manually entered under the Configuration tab on the Unit Types screen. This is generally used when a unit type does not have any true competitors in the area.

Market Composite Option

Indicates whether or not the market composite floor is used when calculating reference rent. This is determined by whether or not the Use Floor box is checked on the Reference Rent screen.

Business Case

The DRRC formula that is being applied in order to determine the reference rent amount. There are several different business formula options that can be triggered based on the values of five different components (Exposure, Leasing Velocity, Premium Current, Comps Above/Below, and Comps Increasing).

Normal Exposure

Displays whether an exposure percentage is considered low, medium, or high based on the percentages set for parameters 71 (Threshold Low Exposure) and 72 (Threshold High Exposure). These parameters can be set at either the community level or at the unit type level under Admin>Parameters.

Premium Current

Indicates whether the last reference rent was higher than the last market composite.

Comps Above or Below

Indicates whether the current market composite is above or below the last reference rent value.

Comps Increasing

Indicates whether the current market composite is above the last market composite.

Leasing Velocity Value

The weighted percentage of move-ins compared to either guest cards or exposed units (whichever value is lower) over the last five weeks. More information can be viewed under LRO>Pathfinder>DRRC LV Calc.

Leasing Velocity, Normalized

Shows what range the leasing velocity percentage falls within (low, medium, or high) based on the percentage settings of parameters 107 (LV Threshold High) and 108 (LV Threshold Low). These parameters can be set at either the community level or at the unit type level under Admin>Parameters.

Exposure

Percentage of units available within the unit type in the next number of days specified by parameter 153 (Exposed Units Range). The default number of days is 60. This parameter is configurable at the community level under Admin>Parameters>Community.

Available Units

The number of units available within the unit type in the number of days specified by parameter 153 (Exposed Units Range). The default number of days is 60. This parameter is configurable at the community level under Admin>Parameters>Community.

Total Units

The total number of units in the unit type.
