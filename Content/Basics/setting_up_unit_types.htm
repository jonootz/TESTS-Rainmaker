Setting Up Unit Types

Unit Types are set up in the system and mapped to unit types from the PMS system. When a new community is created, unit types are set up with a 1-to-1 mapping for each PMS unit type. When the nightly processes are run, if a new PMS unit type is imported, a new unit type is created and mapped to it. By default, the unit types have the same names as the PMS unit types.

PMS unit types may have lowercase and uppercase letters. unit types can only have uppercase letters. So a PMS unit type named 123b would be mapped to a unit type named 123B.

If multiple PMS unit types have the same name, the system will still create a 1-to-1 mapping with each unit type. The second unit type will have an "&1" on the end, the third unit type will have an "&2" on the end, and so on. Learn about the Unit Types screen

To set up and maintain unit types:

  1. Log into the Web UI and select the Configuration tab in the top title bar.

  2. Select the community.
  3. Select the Unit Types screen. This is where you'll set up and maintain the unit types for the community.

Adding a Unit Type

  1. On the Unit Types screen, click Add Unit Type.
  2. Enter a name or ID for the unit type and a description.

  3. Click OK.

    Next you'll need to edit the unit type to get it set up how you want it, including assigning it to the correct unit category.

Editing a Unit Type

After you've created a unit type, you can edit the description, market composite floor, move in cost, make ready days, renewal cost, unit category, number of beds, and number of baths. If the selected unit category is not optimized, you can edit the Exclude from Billing setting.

Beds and Baths information is used when creating competitor unit types (which are based on your unit types).

  1. On the Unit Types screen, click the pencil icon on the row for the unit type you want to edit.
  2. Make your changes.
  3. Click the save icon on the unit type row to save your changes. If you don't want to save your changes, click the cancel icon .

After you set up a new unit type, you'll need to add competitor mappings, weights, and positions for the new unit type before running the nightly processes. This information is added under LRO>Competitors.

You should run the nightly processes after adding a new unit type or assigning a unit type to a new unit category.

Deleting a Unit Type

You can delete unit types that don't have any PMS unit types mapped to them.

  • To delete all unit types that aren't being used, click Remove Unused Unit Types on the Unit Types screen.

    This will remove all unit types that aren't mapped to PMS unit types.

  • To delete a specific unit type, click the delete icon on the row of the unit type you want to delete. This icon appears next to the pencil icon. If you don't see the delete icon, then the unit type cannot be deleted.

Renaming a Unit Type

You can't edit a unit type's name. You have to create a new unit type and delete the old one.

Unit type names cannot contain spaces.

  1. On the Unit Types screen, add a new unit type with the name you want.
  2. Set the unit type up the same way as the original unit type.
  3. On the PMS Unit Types screen, change any PMS unit types that were mapped to the original unit type to be mapped to the new unit type.
  4. On the Unit Types screen, click the delete icon on the original unit type row. This icon appears next to the pencil icon if the unit type can be deleted.
