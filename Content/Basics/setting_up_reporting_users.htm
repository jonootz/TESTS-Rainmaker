Setting Up Users With Reporting Access

The Business Intelligence reports are located under the Reporting tab. Setup for the Business Intelligence reports is under the Reporting Admin tab.

Following is how you set up users in the LRO system to have access to the Reporting tab and/or the Reporting Admin tab.

Required for Access

To access the Reporting tab, a user must have the following:

  • a valid email address
  • the REPORTING.MODULE right
  • the REPORTING Report Access All Properties right

If a user is part of a rights group with REPORTING Report Access All Properties, they will have the last two items on the list above.

If a user should have access to the Reporting Admin tab, they should also be given the REPORTINGADMIN.ReportingAdmin right.

Updating Group Rights

We recommend that user rights be managed at the Group level. This allows user rights changes to be made quickly and consistently across all users assigned to each Group.

Setup for Rights Group

To set up Reporting access for a Rights Group (such as CORP, OPERATOR, or MANAGER):

  1. Go to Admin>Users.
  2. Click the pencil icon on the line for the Group you want to update.
  3. Check the box for REPORTING.ReportsAllProps.
  4. Click Save.
  5. Make sure all users assigned to this Group have an email address entered. If they don't have an email address entered, they won't be able to access the Reporting tab even if they have the correct rights.

These changes will take effect the next day. Users who are assigned to this Group will then have access to the Reporting tab.

Reporting data will only be available for the properties that users are assigned to in this Rights Group.

Reporting Admin access cannot be assigned at the Rights Group level. It must be done at the Full Group or Individual level.

Setup for Full Group

To set up Reporting and/or Reporting Admin access for a Full Group (such as ADMINISTRATOR or PRICINGANALYST):

  1. Go to Admin>Users.
  2. Click the pencil icon on the line for the Group you want to update.

  3. For access to the Reporting tab, check the box for REPORTING.MODULE.

    For access to the Reporting Admin tab, check the box for REPORTINGADMIN.ReportingAdmin.

  4. Click the Property Rights link.

  5. If the Group should be able to view all properties under Reporting, check the All Props box for REPORTING Report Access All Properties.

    If the Group should only be able to view certain properties under Reporting, select those properties in the Properties list on the left side of the screen. Then check the Selected box to the right of REPORTING Report Access All Properties.

  6. Click OK.
  7. Click Save.
  8. Make sure all users assigned to this Group have an email address entered. If they don't have an email address entered, they won't be able to access the Reporting tab or Reporting Admin tab even if they have the correct rights.

These changes will take effect the next day. Users who are assigned to this Group will then have access to the Reporting tab and/or the Reporting Admin tab.

Users in this Group can view reporting data will be available for all properties unless you selected specific properties in step 5 above.

Updating Individual Rights

If a user doesn't fall into a Group, you can set up their Reporting and/or Reporting Admin access at the individual user level.

  1. Go to Admin>Users.
  2. Click the pencil icon on the line for the user you want to update.

  3. For access to the Reporting tab, check the box for REPORTING.MODULE.

    For access to the Reporting Admin tab, check the box for REPORTINGADMIN.ReportingAdmin.

  4. Click the Property Rights link.

  5. If the user should be able to view all properties under Reporting, check the All Props box for REPORTING Report Access All Properties.

    If the user should only be able to view certain properties under Reporting, select those properties in the Properties list on the left side of the screen. Then check the Selected box to the right of REPORTING Report Access All Properties.

  6. Click OK.
  7. Click Save.
  8. Make sure the user has an email address entered. If they don't have an email address entered, they won't be able to access the Reporting tab or Reporting Admin tab even if they have the correct rights.

These changes will take effect the next day. This user will then have access to the Reporting tab and/or the Reporting Admin tab.

This user can view reporting data will be available for all properties unless you selected specific properties in step 5 above.

Troubleshooting Reporting Access Setup

I set up the user's reporting access and ran a nightly process, but the user is getting an error message when they click on the Reporting tab and/or the Reporting Admin tab.

The task that updates user rights for Reporting and Reporting Admin runs once daily and is not associated with any property. It is necessary to wait one day for the user rights update to take effect.

It has been a full day since the user's reporting access was set up, but the user is getting an error message when they click on the Reporting tab and/or the Reporting Admin tab.

The user is probably missing one of the three required items for Reporting/Reporting Admin access. Check that the user has been set up with the required items or that they are assigned to a Group with the necessary rights and have an email address entered.

A user who should only be able to see reporting data for certain properties can see reporting data for the entire portfolio.
