Unit Types

When a new community is created or new PMS unit types are imported, Unit Types are created with a 1-to-1 mapping for each PMS unit type. You can change the unit types and the PMS mappings, if necessary. PMS mappings are changed on the PMS Unit Types screen.

In the Web UI, go to Configuration>Unit Types to set up the unit types and enter information about the unit types that the system uses when calculating prices.

Understanding the Unit Types List

Unit Type - This is the name or ID of the unit type. This is entered when the unit type is added and cannot be changed. For unit types that are created as part of new community setup or when new PMS unit types are imported, the name matches the name of the mapped PMS unit type.

Description - This is entered when the unit type is added. You can change it if you want to. For unit types that are created as part of new community setup or when a new PMS unit type is imported, the description matches the description of the mapped PMS unit type.

Avg Sqft - This is the average square feet of the units in the unit type. This is calculated by the system using a weighted average. The square footage amounts used in the calculation come from the PMS system.

For new unit types, this field defaults to blank. It will be populated after the extractor runs, which is typically overnight.

For unit types that are created as part of new community setup, this field is blank until the extractor is run a second time. This gives you a chance to make any changes to the unit types and mappings after initial setup.

You cannot edit this field. It is important that square footage amounts are entered correctly in the PMS system for this calculation to be correct.

Units - This comes from the PMS system based on the unit type mapping. For new unit types, it will be populated after the extractor is run. For unit types created as part of new community setup, it will be populated correctly after the extractor is run twice. You cannot edit this field.

Market Composite Floor - This defaults to $0.00. You can change it if you want to. This would typically be used during the time you are getting your competitors set up or if the competitive data is untrustworthy.

The low boundary for pricing comes from either the market composite or the market composite floor. The market composite comes from your competitor information. If you have a small or limited competitive set or need to set the low boundary to meet a specific business goal, you can enter a market composite floor amount here.

In the Web UI on the Reference Rent screen (under the LRO tab), there is a Use Floor setting for each unit type. If this is checked, the market composite floor is used as the low boundary for pricing. If it's not checked, then the market composite is used as the low boundary for pricing.

Move In Cost - This defaults to $500.00. You can change it if you want to. This amount is used by the system to calculate the price. It represents the standard turn cost for that unit type. You can adjust this amount to properly reflect the turn costs for the unit type.

Make Ready Days - This defaults to 0. You can change it if you want to. This is the number of days needed to get the unit ready to rent. It is used to calculate the available date if you don't get the available date from the PMS system. You would change this if the available date is not set in the PMS system; otherwise leave it set to 0.

Renewal Cost - This defaults to $100.00. You can change it if you want to. This is used to calculate renewal pricing. It represents the average cost tied to a lease renewal. It is usually not changed.

Unit Category - You should select the unit category for this unit type. By default, this is set to the 0x0 unit category. The 0x0 unit category is not optimized and prices won't be generated for unit types assigned to it. Unit categories are set up on the Unit Categories screen.

Beds - This is the number of bedrooms for the unit type. This information is used when creating competitor unit types (which are based on your unit types).

Baths - This is the number of baths for the unit type. This information is used when creating competitor unit types (which are based on your unit types).

PMS Unit Types - This shows which PMS unit types are mapped to this unit type. You can have multiple PMS unit types mapped to one unit type. You cannot edit this field. PMS unit types are mapped to unit types on the PMS Unit Types screen.

Exclude From Billing - This defaults to No which means the customer will be billed for pricing the unit type. If this unit type is assigned to a unit category that is set to Optimize, it cannot be excluded from billing.

Action Column - The last column contains the icons for actions you can take on each unit type. Options are Edit , Delete , Save , and Cancel . Save and Cancel are available after you click Edit. Delete is available if the unit type is not mapped to any PMS unit types. You must enable editing for these options to appear in the column.

Turn costs (Move-In Cost and Renewal Cost) do not include vacancy days. Costs associated with lost rent are calculated by the system based on the community's history.

Understanding Other Unit Type Options

Add Unit Type - Opens a dialog box where you can add a unit type name and description to create a new unit type. You must enable editing to see this option.

Remove Unused Unit Types - Deletes any unit types that aren't mapped to PMS unit types. You must enable editing to see this option.

Export to Excel - Select the Excel icon in the upper right corner of the screen to open the unit type list in Excel.
