Renewal Pricing Report

The Renewal Pricing report shows the renewal rents for units that are expiring within a certain time period. This report is typically run once a month for a future time frame. For example, if it is the beginning of February, you might run the report for leases expiring in the month of April. You may run the report more often or for earlier or longer time frames depending on your renewal strategy and business processes.

Using the Renewal Pricing Report

You'll use this report to see your renewal pricing for leases expiring in a particular time period. You can also view information about the expiring leases, see the parameter settings used to generate the renewal pricing, and print renewal letters from this screen.

Viewing the Renewal Pricing Report

You'll find the Renewal Pricing report under LRO>Renewal Pricing.

The Summary view of the report shows one line per resident with the best renewal lease term information.

The Detail view of the report shows all lease terms for each resident along with current lease information and the parameter settings used in renewal pricing for that unit. The best renewal lease term will be shown in bold text. This view is useful for negotiating renewal lease terms and rents with current residents.

The Mgmt Report button opens a summary view of the report with some statistics displayed at the top regarding renewal rate increases and decreases. This is often used by corporate teams to quickly review the renewal offers before they are sent out.

The Mail Merge button opens an Excel spreadsheet with the renewal information included so it can be used with Microsoft Word's mail merge functionality to create renewal letters.

Understanding the Information on the Renewal Pricing Report

The current resident's name and unit ID are shown to identify each lease that is expiring within the specified time period.

Lease/Unit Information

Current Perceived Rent: This is the rent that the resident is currently paying

Amenities: Description of any amenities for this unit. The PMS unit type and pricing offset amount are shown in parentheses.

Amen Amt: This is the amount being charged for any amenities. This is included in the current perceived rent amount.

Current Lease Term: The length of the current lease.

Current Expiration Date: The date the current lease expires.

Renewal Date: The first day of the new lease.

Rent Increase: The difference between the Effective Rent for the best renewal lease term (in bold) and the Current Perceived Rent. This is shown as a dollar amount and a percentage. If the amount and percentage are negative, it means the renewal rate is less than what the resident is currently paying.

Discount to Market: This is the difference between the Effective Rent and the Move In Rate for the lowest priced renewal lease term (bold). This is shown as a dollar amount and a percentage. If the amount and percentage are negative, it means the renewal rate for the resident is less than what a new resident would pay for that unit for the same lease term.

Rate: This is the Increase Rate percentage for this unit type which specifies by how much of the difference between the resident's current rent and the new move-in rent should the resident's rent be increased. This parameter is set under Admin>Parameters>Renewal Config.

Min Increase: This is a minimum percentage that the resident's rent must increase based on their current rent. This parameter is set under Admin>Parameters>Renewal Config.

Max Increase: This caps the amount that the resident's rent can increase based on their current rent. This parameter is set under Admin>Parameters>Renewal Config.

Renewal Pricing Information

Lease Term: This is the number of months that the lease is for.

Effective Rent: This is the renewal rate for this unit.

Base Rent: This is the same as the effective rent unless a concession is in place.

Total Concession: This is the amount of any concessions that are being offered for this unit. This is determined by parameter 136 - Concession Offer Factor.

Move In Rate: This is the rate that a new resident would pay for this unit if it was available to lease.

Discretionary Concession: This is rarely used. It gives the on-site team the ability to negotiate the renewal offer based on the amount of the concession. This is typically handled as a one-off situation and not configured in the system. It is not factored into any of the offers. It applies to residents who have Current Perceived Rents that are greater than the new Move In Rate for the unit. If you want to have a discretionary concession, you would use parameter 135 - Discretionary Concession Factor.
