Seasonality Graph

The Seasonality Graph shows you the seasonality for a particular unit category for the past 12 months. By default, the system calculates seasonality based on three years of data. You can adjust the seasonality that has been calculated, if necessary.

How does the system use seasonality?

Viewing the Seasonality Graph

The Seasonality Graph is found under LRO>Charts>BI Charts. Select Seasonality from the Report drop-list. Then you can select the unit category that you want to view or adjust the seasonality for.

The seasonality that has been calculated by the LRO system is shown in blue. The seasonality that the system is using is shown in red. This will be the same as the calculated seasonality unless you have changed it.

Adjusting Seasonality on the Seasonality Graph

In general, it is best to use the calculated seasonality. However, there may be circumstances where you feel like the seasonality should be adjusted. If you are not sure if you should adjust your seasonality, then you shouldn't change it.

Why would I adjust my seasonality?

To change the seasonality that the LRO system is using, click the Edit button located in the bottom right corner of the screen.

If you don't see the Edit button, then you do not have access to change the seasonality.

You can change the seasonality values in the table or you can move the line on the graph.

Adjusting Seasonality on the Graph:

Each circle represents the seasonality for a week.

  1. Find a week you want to adjust.
  2. Click on the circle for that week.
  3. Drag it to the new seasonality number.
  4. Repeat for each week you want to adjust.

  5. Click Save to save the new seasonality.

    In order to save the new seasonality, the "Adjustment to be made" must equal 0. The line on the graph will be green.

    After saving, the adjusted seasonality will appear as the red line on the graph. This is the seasonality that the system will use in its pricing calculations.

Adjusting Seasonality in the Table:

Each row represents the seasonality for a week and each column represents a month.

  1. Find a week you want to adjust.
  2. Enter the new seasonality amount in the cell for that week in a particular month.

  3. Repeat for each week you want to adjust.

    If you check the "Change monthly values" checkbox located below the table, you can enter a value for one week in a month and that same value will populate all weeks for that month. This only applies to the values in the table. It does not affect moving seasonality on the graph.

  4. Click Save to save the new seasonality.

  5. In order to save the new seasonality, the "Adjustment to be made" must equal 0. The line on the graph will be green.

    After saving, the adjusted seasonality will appear as the red line on the graph. This is the seasonality that the system will use in the pricing calculations.

Tips

When you change the seasonality, the "Adjustment to be made" must equal 0. After you make changes, you'll see the amount of any adjustments that must be made before you can save the new seasonality. The adjustments must balance out. For example, if you increase the seasonality for a week by .1, you must decrease the seasonality by .1 in another week or weeks.

The seasonality for each week must be between the Min Seasonal Factor (parameter 19) and the Max Seasonal Factor (parameter 20). These limits determine the range on the Y-axis of the graph.

The LRO system will continue to calculate seasonality when the UPSTAT process is run. However, if you make changes to the seasonality on the Seasonality Graph, the edited seasonality will be used in the pricing calculations.

Seasonality Outside of Limits

If you set parameter 19 to be greater than the smallest seasonality value or set parameter 20 to be smaller than the largest seasonality value and the nightly processes have not been run, you will get an error message on the Seasonality Graph screen. You will not be able to edit the Seasonality Graph unless you change the parameters or until the nightly processes have run.
