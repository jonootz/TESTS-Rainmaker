Business Intelligence Reporting Admin and Setup: Reporting Groups

A reporting group (also known as a hierarchy) determines the information that appears on many of the reports and how that information can be filtered.

Prior to release 14.11, you could set up a reporting hierarchy from the Admin tab under Communities>Hierarchy and if you had one set up there, it would initially show up as a reporting group under the Reporting Admin tab. Now, you can only set up hierarchies under the Reporting Admin tab.

The reporting group is a group of attributes. Attributes values are assigned to your properties. When you view some reports, you will select the reporting group you want to use. The reporting group determines which properties are included on the report and how you can filter the information.

Make sure you give everyone using the reports a list of the attributes that make up each reporting group (in the correct order). Some reports can be filtered by levels which correspond to the order of the attributes in a reporting group. The report will just show "Level 1", "Level 2", etc. so you need to make sure everyone knows what each level is.

View Reporting Groups

To view your reporting groups, go to Reporting Admin>Reporting Group.

Add a Reporting Group

  1. On the Reporting Groups screen, click Add Reporting Group.

  2. On the Add Reporting Group dialog, enter the name for the reporting group.

  3. Click OK.

    Now you'll need to add attributes to the reporting group.

Add Attributes to a Reporting Group

  1. On the Reporting Groups screen, click the name of the reporting group you want to add attributes to.

  2. On the Reporting Group Details screen, click Add Attributes.

  3. Check the checkbox next to each attribute you want to include in the reporting group.

    The maximum number of attributes you can have in a reporting group is 10. If you select more than 10, only the first 10 in the group will be used.

  4. Click OK.

Edit a Reporting Group's Name

  1. On the Reporting Groups screen, click the Edit button on the line for the reporting group you want to edit.

  2. Change the reporting group name on the Edit Reporting Group dialog.

  3. Click OK.

Remove Attributes from a Reporting Group

  1. On the Reporting Groups screen, click the name of the reporting group you want to edit.

  2. Click the Remove button on the line of the attribute you want to remove.

  3. Click OK on the message confirming you want to remove the attribute.

When you remove an attribute, the sort order for the remaining attributes is not reordered. For example, if you have 4 attributes and you remove the third one, the sort order will show 1,2,4. When you view reports that use hierarchies, the attribute with sort order 4 will be level 3 so there is no gap between levels. If this happens, you can change the order of the attributes to make sure they match how they appear on the reports.

Change the Order of the Attributes in a Reporting Group

The sort order of the attributes for a reporting group determines the hierarchy levels when you are viewing a report that uses hierarchies. For example, the attribute with number 1 under sort order will be level 1 when you view reports that use hierarchies.

To change the order of your attributes:

  1. On the Reporting Groups screen, click the name of the reporting group you want to edit.

  2. Click the drop-down arrow next to the Remove button on the line of the attribute you want to move in the list.

  3. Select Move Up or Move Down to move the attribute to the correct location in the list.

  4. Repeat steps 2 and 3 for each attribute you want to move in the list.

Delete a Reporting Group

  1. On the Reporting Groups screen, click the drop-down arrow next to the Edit button on the line for the reporting group you want to delete.

  2. Select Delete.

  3. Click OK on the message confirming that you want to delete the reporting group.

If you only have one reporting group and you delete it, you will not see any data on your reports that use hierarchies.

Deactivate/Activate a Reporting Group

When you add a reporting group, it is automatically set to be Active. If you don't want to use a reporting group, but you don't want to delete it, you can make it inactive.

To deactivate a reporting group:

  1. On the Reporting Groups screen, click the drop-down arrow next to the Edit button on the line for the reporting group you want to deactivate.

  2. Select Deactivate.

  3. Click OK on the message confirming that you want to deactivate the reporting group.

If you only have one reporting group and you deactivate it, you will not see any data on your reports that use hierarchies.

To reactivate a reporting group:

  1. On the Reporting Groups screen, click the drop-down arrow next to the Edit button on the line for the reporting group you want to activate.

  2. Select Activate.

  3. Click OK on the message confirming that you want to activate the reporting group.

Troubleshooting
