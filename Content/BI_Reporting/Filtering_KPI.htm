Filtering the KPI Report (BI Reporting)

There are two different views of the report, an Overview and a Detail view. Some of the filters for these views are independent of each other. For example, if you have the Overview filtered for a particular community and then switch to the Detail view, the Detail view will not automatically be filtered for that community.

You can use the Input Controls to filter the report and control how the information is displayed.

Tip

When you open the report, you will be prompted to select a hierarchy. If you want to view the report using a different hierarchy, you can close and reopen it or select the refresh button.

Level of Detail to Display (Detail View only)

If you choose to view the report at the Hierarchy & Community or Hierarchy, Community & Unit Type levels, there will be a large amount of data on the report so you may want to filter the report down to a specific community and/or unit type, especially if you plan on printing the report.

Level to Display

By default, the KPI report displays information for Level 1. For example, you may have a reporting group set up with Corporate as Level 1. The Corporate attribute has one value called Corporate and it is applied to all properties. This allows you to roll up information at the corporate level and then use the Level to Display filter to display the information broken out by other attributes. For example, if Level 4 is State, if you select to display Level 4, the report will be shown with the information broken out by state.

The Level to Display filter is useful when no other filters are selected or if you have selected multiple communities. It is not very useful if you have selected one filter item. For example, if you have selected one community in the community filter, the report will only show information for the selected community at every level you select.

When you view the report by level, each value for the level is shown on its own page. For example, if you selected Level 4 which is State, each page displays the report by a different state. You can see the attribute value in the filter box next to Hierarchy and Level Value.

Use the Level to Display drop-list to select the level you want to display.

Chart Options

By default, the KPI report information is broken out by month. You can choose to break it out by day, week, or quarter, if you want to.

If you choose to view the report broken out Daily, there will be a large amount of data on the report so you may want to filter the report down to a specific community and/or unit type before choosing to view the information Daily.

Display Measures (Overview only)

This allows you to show and hide the following average rent measures on the Overview graph:

  • Average Rent Expiring Leases
  • Average Rent New Leases
  • Average Rent Renewing Leases

Occupancy and Average Rent Leases in Place will always appear on the graph.

Filter Dates >=

This is the lower boundary of the date range. Use this filter to show effective rent and occupancy information on or after a particular date. If you only enter a date here, the report will display information from that date on. If you want to set a range, you would enter the upper boundary in the following filter (Filter Dates <=).

Filter Dates <=

This is the upper boundary of the date range. Use this filter to show effective rent and occupancy information on or before a particular date. If you only enter a date here, the report will display information from that date and earlier. If you want to set a range, you would enter the lower boundary in the previous filter (Filter Dates >=).

If you set the Filter Dates to the same date, you won't see anything on the Overview. This is because the graph displays a line and you can't create a line with a single date. You can see the information for a single date on the Detail view.

Community (ID)

The Community filter allows you to run the report for all communities, multiple communities, or a specific community.

By default, the report includes either all communities that you have access to that use an attribute in the selected hierarchy or the communities you selected on the Prompts dialog when you opened the report.

Unit Type

You can filter the report to only show data for a particular unit type or selection of unit types.

The unit type filter will only contain unit types that apply to the communities that were selected in the Prompts window. If you didn't select any communities in the Prompts window, the report was run for all communities and all unit types will be in the Unit Type filter.
