Audit Hierarchy Report (BI Reporting)

The Audit Hierarchy report details the reporting hierarchies that have been set up and the communities that are assigned attribute values within each hierarchy.

Viewing the Report

The report will open when you click the Audit Hierarchy link on the Reporting tab. There are two tabs on the report, Hierarchy and Unassigned.

Hierarchy Tab

The Hierarchy tab shows you all the details about the hierarchies that have been set up and which communities have been assigned attribute values for attributes within the hierarchy and what those attribute values are.

You can filter this tab using the Input Controls.

Reporting Group Name Filter

By default, the report will include all reporting groups you have set up. If you have multiple reporting groups set up, you can filter the report to only show a particular reporting group.

Active Filter

By default, the report will include both active and inactive communities. You can filter the report to only show one or the other.

The Is Active column indicates if a community is active or not.

Filter Bar

Another way you can filter the information on the report is using the Filter Bar. The attribute filters will appear right below the toolbar.

There is a filter for each attribute in the reporting groups or selected reporting group. The drop list for each filter contains the values for each attribute. If you select a value, only properties that have that value assigned for that attribute will be included in the report.

If you don't see the Filter Bar, select the Filter Bar button in the toolbar above the report.


Unassigned Tab

The Unassigned tab shows you a list of any communities that have not been assigned an attribute value for attributes in any hierarchies. Unassigned communities will not appear on any reports.

You may want to print this report or export it to Excel or PDF to use as a reference when viewing the reports that use hierarchies.
