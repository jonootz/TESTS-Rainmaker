Troubleshooting Tips for Business Intelligence Reporting

I see N/A as an attribute on some reports.

You'll see N/A on a report if you display the report at a level (attribute) that some or all properties do not have values for. N/A is for information from any properties that meet the filter criteria, but don't have an attribute value assigned for the level being displayed.

You can view some reports at different levels by drilling down on the levels for the report or by selecting the level in the Level to Display input control.

This may happen because an attribute value hasn't been assigned to some or all properties. For example, let's say you have 3 properties: P1, P2, and P3. The second attribute in your reporting group is Region. You have assigned a region value of Northwest to P1, nothing for P2, and Southeast for P3. If you select to view the report at level 2 (region), you'll see information for Northwest, infomation for Southeast, and information for N/A. Information for P2 will be in the N/A part.

This could also happen if you display the report at a level that the reporting hierarchy doesn't have. For example, a hierarchy can have up to 10 levels. If the reporting hierarchy only has 4 levels in it, the information on the report will show as an attribute called N/A if you select to view the report for levels 5-10.

I selected a Community or Unit Type and now my report doesn't show any information.

  • Make sure the community or communities selected are included in the hierarchy that is applied.

  • Make sure the unit type or unit types selected exist in the community or communities that are selected.

    You can look in the Filter Box to see which communities and unit types are valid selections based on other selections that have been made. For example, if Unit Type Applied shows 1X1 and 2X2, but you select 2X1 in the Unit Type drop list, the report will not show any information.
