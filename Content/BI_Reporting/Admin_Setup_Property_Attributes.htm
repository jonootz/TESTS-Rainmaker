Business Intelligence Reporting Admin and Setup: Property Attributes

On the Reporting Admin Property Attributes screen, you can add, edit, and delete attributes. Attributes are used to make up the reporting groups (hierarchies) you want to use for your Business Intelligence reports.

Prior to release 14.11, you could set up a reporting hierarchy from the Admin tab under Communities>Hierarchy and if you had one set up there, the property attributes set up there would initially show up here.

How Are Property Attributes Used

The property attributes are used to help you view your reporting information in ways that are beneficial to you. For example, you can view information at the corporate level which will include all of your properties, but then you can also view the information at the regional level or the state level if you have set up region and state attributes. The attributes will have values, for example, different regions such as Southeast and Northwest and different states such as Georgia and Oregon. You will assign the appropriate attribute values to your properties so the correct information is included when you view the information at different levels on the reports.

Property attributes are set up here and then assigned to properties.

To add a property attribute:

  1. In the Web UI, go to Reporting Admin>Property Attribute.

  2. Click Add Attribute.

  3. Enter a name and description for the attribute.

  4. Click OK.

The attribute will now be in the list. In order to use this attribute, it needs to have values.

To add values for an attribute:

  1. On the Property Attributes screen, click the name of the attribute.

  2. On the Property Attribute Details screen, click Add Value in the Attribute Values section.

  3. Enter the value.

  4. Click OK.
  5. Repeat steps 2-4 for each value you want to add.

Now you can assign these values to your properties.

  1. In the Properties section, click Edit on the row for each property that you want to assign a value for this attribute.

  2. Select the attribute value for each of the properties.

  3. Click Save on each row to save the value for each of the properties.

You'll need to go into the Reporting Group screen and add this attribute to your reporting groups in order to filter by it on the reports.

To edit a property attribute:

To change the name or description of a property attribute:

  1. On the Property Attributes screen, find the property attribute you want to edit in the list.

  2. In the Actions column, click the Edit button.

  3. Edit the name and/or description.

  4. Click OK.

The property attribute stays in the same reporting groups and is still assigned to the same properties with the new name.

To change the name of a value for an attribute:

  1. On the Property Attributes screen, click the name of the attribute that has the value you want to change.

  2. On the Property Details Attribute screen, find the value you want to change in the Attribute Values section.

  3. In the Actions column, click the Edit button.

  4. Change the name of the value.

  5. Click OK.

The value is still assigned to the same properties with the new name.

To delete a value for an attribute:

  1. On the Property Attributes screen, click the name of the attribute that has the value you want to delete.

  2. On the Property Details Attribute screen, find the value you want to delete in the Attribute Values section.

  3. In the Actions column, click the drop-down arrow next to the Edit button for that value.

  4. Select Delete.

  5. Click OK on the message confirming you want to delete the attribute value.

The attribute value will be removed from any properties that it was assigned to and can no longer be used to filter the reports. (These changes will take effect after the nightly processes are run.)

To delete a property attribute:

  1. On the Property Attributes screen, find the property attribute you want to delete in the list.

  2. In the Actions column, click the drop-down arrow next to the Edit button for that attribute.

  3. Select Delete.

  4. Click OK on the message confirming you want to delete the attribute.

The attribute will be removed from any properties and reporting groups that it was assigned to and can no longer be used to filter reports. (These changes will take effect after the nightly processes are run.)

Troubleshooting

I created a new attribute, but I don't see information from properties that should have this attribute and/or I can't filter the report using the new attribute.

There are several reasons that this could occur:

  • The attribute wasn't added to a reporting group.
  • Values for the attribute weren't assigned to any properties or to the correct properties.
  • The report may be filtered so that the information is not being included. Be sure that you have selected a reporting group that includes this attribute (if applicable) and that your other filters would not exclude the information you are expecting to see. Also make sure a value for this attribute is assigned to the properties you are expecting to see information for.
