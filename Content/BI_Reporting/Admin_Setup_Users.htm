Business Intelligence Reporting Admin and Setup: Users

On the Reporting Admin Users screen, you can add and edit the roles that users have concerning the reports under the Reporting tab.

User information comes from the database as part of the nightly processes. Users that appear under the Reporting Admin tab have REPORTING.MODULE rights and REPORTING Report Access All Properties property rights and/or REPORTINGADMIN.ReportingAdmin rights. They also have valid email addresses.

In the Web UI, go to Reporting Admin>User.

Roles

Admin: This role gives users access to view and change information on the Reporting Admin tab. Most users will not have this role.

MGT Content Viewer: This role allows users to view the management reports and dashboards under the Reporting tab.

Pricing Revenue Manager: This role is for users who are responsible for pricing and revenue management in their organization. Currently, assigning this role to a user won't do anything.

This screen was set up to have the option for more roles in the future. This screen should currently be used to see which users can access the Reports and/or Reporting Admin tab. Learn how to set up users with Reporting access

Properties and Property Groups

Properties and property groups are assigned to customer users under the Admin tab. You can see which properties and/or property groups are assigned to a user by viewing the user's details.

Troubleshooting

I don't see a user on the Users screen.

Possible Reasons:

  • The nightly processes haven't run since the user was set up or updated.
  • The user doesn't have a valid email address.
  • The user hasn't been set up with the necessary rights.

Users come over from the database as part of the nightly processes. If you set up a user under the Admin tab, it should show up on the Users screen under the Reporting Admin tab the next day.

In order for users to appear on the Reporting Admin Users screen, they must be set up with at least one of the following rights: REPORTINGADMIN.ReportingAdmin, REPORTING.MODULE, REPORTING.* property rights (REPORTING Report Access All Properties). Users also must have a valid email address.
