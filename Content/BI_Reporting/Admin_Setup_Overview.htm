Business Intelligence Reporting Admin and Setup: Overview

There are two places in the Web UI where you set up information for using Business Intelligence reporting. Some information is set up under the Admin tab and some is set up under the Reporting Admin tab.

Setup

  1. Set up users under the Admin tab on the Users screen with access to REPORTING.MODULE and REPORTING Report Access All Properties property rights. If you want a user to have access to the Reporting Admin tab, you will need to give them access to REPORTINGADMIN.ReportingAdmin as well. Make sure the users have valid email addresses, also.

     

  2. Under Admin>Users, you'll also set up properties and property groups and assign them to users.

  3. Users and property information will be brought over into the Reporting Admin tab as part of the nightly processes. After the users and property information appear under the Reporting Admin tab, you can continue with the setup in the Web UI.

  4. On the Reporting Admin Users screen, verify that your users have the correct roles assigned to them. The users should already have the MGT Content Viewer role. This allows them to see the reports on the Reporting tab. If you want a user to have access to view and change information on the Reporting Admin tab, make sure they have the Admin role by giving them the REPORTINGADMIN.ReportingAdmin right from the Users screen under the Admin tab.

  5. On the Property Attributes screen, make sure the attributes are the ones you want to use for the Business Intelligence reports.You can add, edit, and delete property attributes here.

  6. On the Property Attribute Details screen, verify that your attribute values are correct. You can add, edit, and delete attribute values and assign attributes to properties here.

  7. On the Properties screen, verify the correct attributes are assigned to your properties. You can assign attributes to a property or change the assigned attributes by clicking the link in the PMS Prop ID field, and then clicking Edit on the Property Details screen.

  8. On the Reporting Groups screen, you'll set up your reporting groups. You must have at least one reporting group. When you view certain reports, you'll be able to select which reporting group you want to use.

    Prior to release 14.11, you could set up a reporting hierarchy from the Admin tab under Communities>Hierarchy and if you had one set up there, it would initially show up as a reporting group here. Now, you can only set up hierarchies under the Reporting Admin tab.

When Changes Will Take Effect

Changes that you make on the Reporting Admin tab and user setup under the Admin tab will take effect after the nightly processes are run. So basically, after you make any changes to your Business Intelligence Admin setup, including adding users, you'll have to wait until the next day for them to take effect.
