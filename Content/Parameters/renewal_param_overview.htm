Renewal Parameters

The renewal parameters help you communicate your business strategy regarding renewals to the system. These parameters can be modified to accommodate almost any renewal strategy.

The renewal parameters are often changed seasonally. More aggressive renewal offers are presented during peak seasons with more conservative offers during slow seasons. The renewal parameters are also changed if the community strategy shifts or if there has been a significant increase or decrease in asking rents for new leases.

You should review and approve your renewal batches monthly to make sure your renewal parameter settings are working correctly for your renewal strategy.

Following are the renewal parameters you'll be most likely to change.

Renewal Config Parameters Screen

The Renewal Config Parameters screen is where you enter Drop/Increase Rates, Minimum Increases, and Maximum Increases for different ranges based on the percentage difference from what the resident is currently paying and the new lease pricing for that unit.

These parameters work with the Tier parameters.

By default, you will have three ranges set up. Most likely, you will not need to add new ranges.

From (%) and To (%): This is the range of percentage differences that the parameters are set for. You'll enter this information if you are adding a new range.

Rate (%): This is the % of the difference between the current rent and the tier-adjusted market rent that will be added or subtracted from the renewal offer. For current rents below the tier-adjusted market rent, this is the increase rate. For current rents above the tier-adjusted market rent, this is the drop rate.

Min Increase (%): This is the minimum renewal rent increase or a cap on the drop rate. This value is a percentage. If this is set to 0%, it means no drop in current rent is allowed.

Max Increase (%): This is the maximum renewal rent increase. This value is a percentage. If this is set to 0%, it means that no increase in current rent is allowed.

The Generate Renewals button allows you to easily re-run renewals after changing the renewal config parameters.

Things to note about this screen:

  • A minimum of 3 ranges in required.

  • The zero range will always be included.
  • You must have at least one range above the zero range and one below.
  • The ranges will be continuous. For example, the upper limit (To) of a range will match the lower limit (From) of the range above it.
  • The maximum number of ranges you can have is 21.

Understanding the Range Boundaries

Since the To value from one range will match the From value of the range above it, you may wonder which parameters are used when the percentage equals that value.

  • If the value is 0, it is included in the 0 to 0 range.
  • If the value is a positive number, it will be included in the range where it falls in the To column.
  • If the value is a negative number, it will be included in the range where it falls in the From column.
