243

Maximum Price Level UC Propagation Method

Determines how parameter 41, when defined at the unit type level, will be propagated to the unit category level for optimization.

If parameter 243 is set to Avg, it will average the parameter 41 values for all unit types within the unit category. If parameter 243 is set to Max, it will use the highest parameter 41 value from all unit types within the unit category. If parameter 243 is set to Min, it will use the lowest parameter 41 value from all unit types within the unit category.

Values: Max, Min, Avg

Default: Avg

254

Available Days Param

It specifies whether Vacant Days or Market Days will be used to determine if a unit has historically been a slow or fast mover.

If you want to base the classification off when the unit is actually losing money because it is vacant, you would select Vacant Days. If you are more concerned with the total marketing time, you would choose Market Days.

Values: Vacant Days, Market Days

Default: Market Days

255

Number of Standard Deviations

This is the number of standard deviations about the median which will determine the Fast, Medium, and Slow buckets for the Historical Lease Pace Level.

Generally, the higher number of standard deviations signifies that fewer units will be classified as slow or fast.

The Historical Pace Level buckets may be determined using parameters 274 and 275, if they are set.

Values: >=0

Default: 0.5

Update Pricing

257

Minimum Move Out Observations

This is the minimum number of turnover observations needed for a unit at the current amenity value in order to classify it as a slow, fast, or medium mover. If the amenity value of the unit has been changed recently, the unit will not be classified as slow or fast until it becomes vacant and rented at the current amenity level for the number of times specified by this parameter.

Values: >=0

Default: 2

273

Subtract Vacant Rented and Market Rented Days

This determines if the days the unit was in Vacant Rented status will be subtracted from Vacant Day calculations and if the days the unit was in Vacant Rented or Market Rented status will be subtracted from Market Day calculations.

Sometimes, when a unit comes on market or becomes vacant, it gets rented out but then the prospect cancels causing the unit to go on market or become vacant again. If you would like to exclude the days a unit was in Vacant Rented or Market Rented status from being counted towards the unit's market or vacant days, you should set this parameter to 1.

Values: 0=Don't subtract; 1=Subtract

Default: 1
