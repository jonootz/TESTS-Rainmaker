PMS Unit Type Mapping and Offset Pricing

PMS Unit Type Mapping and Offset Pricing is a feature that allows you to map PMS Unit Types to Unit Types at any time, instead of only during the initial community setup. You can also enter a corresponding Pricing Offset for each unit type, if necessary.

Mapping unit types and entering pricing offsets are typically done by an Business Consultant.

How to use PMS Unit Type Mapping and Offset Pricing

New Communities

  1. In the Web UI, go to the Configuration tab.
  2. Set up unit categories on the Unit Categories screen.
  3. On the Unit Types screen, verify the unit types are correct. Add or edit the unit types as necessary. Assign a unit category to each unit type.
  4. On the PMS Unit Types screen, verify the PMS unit types are correctly mapped to the unit types. Enter the pricing offset amounts for any unit types, if necessary.

Existing Communities

  • You can see the PMS unit type mappings using the Web UI on the Unit Types and PMS Unit Types screens under the Configuration tab. On the PMS Unit Types screen, you can change the mappings if you want to.

  • You can also enter pricing offsets for PMS Unit Types on the PMS Unit Types screen as well. If you previously accounted for an offset as an amenity, you will need to remove it from the amenities after you add it here. Amenities are set up in the PMS system.
  • If you need to set up or change your unit categories or unit types, you can do this on the Unit Categories screen and the Unit Types screen.

You can view and change the PMS unit type mappings and pricing offsets at any time. You'll need to re-optimize or run the nightly processes to see the effect of the changes on pricing.
