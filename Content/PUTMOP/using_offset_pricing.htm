Using Offset Pricing

You'll typically use offset pricing when you have multiple PMS unit types mapped to a single unit type and you want the prices of those units to be different. The system will generate the same price for each unit within an unit type. You can use offset pricing to have adjust the price up or down for a particular PMS unit type.

For example, let's say you have separate PMS unit types for a 3 bedroom/2 bathroom garden apartment and a 3 bedroom/2 bathroom townhouse. Both of these PMS unit types are mapped to one unit type for 3 bedroom/2 bathroom units. The system will generate the same price for both types of units, but you want the townhouse to be $50 more per month than the garden apartment. You can enter a pricing offset to account for this. You can either enter a pricing offset of $-50.00 for the garden apartment or a pricing offset of $50.00 for the townhouse apartment to create a $50 difference in the prices.

Entering Pricing Offsets

Pricing offsets are entered in the Web UI under Configuration>PMS Unit Types. By default, pricing offsets are set to $0.00.

Learn about the PMS Unit Types screen

To enter pricing offsets:

  1. Log into the Web UI and select the Configuration tab in the top title bar.

  2. Select the community.
  3. Select the PMS Unit Types screen.
  5. On the row for the PMS Unit Type that you want to enter a pricing offset for, click the pencil icon .
  6. In the Pricing Offset field, enter the amount of the pricing offset. This can be a positive or negative amount.
  7. Click the save icon .
  8. Repeat steps 5-7 for any additional pricing offsets you want to enter.

If a pricing offset was previously entered as an amenity, you should remove the pricing offset amenity if you enter the pricing offset here, so you don't include it twice. Amenities are set up in the PMS system.

To see the effect of the pricing offsets:

The re-optimization process must be run before the pricing offsets will be included in the pricing reports and the pricing XML. This process is typically run overnight, but you can re-optimize from the Reference Rents screen under the LRO tab in the Web UI to see the effects sooner.

On the New Pricing report, the pricing offset will be included in the Amen Amt value, the Base Rent, and the Effective Rent. The PMS Unit Type and the Pricing Offset amount will appear in parenthesis in the Amenities column.

On the Renewal Pricing report, the pricing offset will be included in the Amenities amount, the Effective Rent, the Base Rent, and the Move In Rate. The PMS Unit Type and the Pricing Offset amount will appear in parenthesis in the Amenities information for each unit in Detail View and in the Amenities column in Summary View. They will also appear in the Amenities column in the Renewal Management Report which is generated from the Renewal Pricing report.
